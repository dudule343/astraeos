📖 Introduction

Conventions de nommage et principes

Comment lire ce dataset, et quelles règles s'appliquent aux données fictives.

🎯 Principe fondateur — La cohérence cross-espaces :

Un même client, cabinet ou ingénieur doit apparaître à l'identique dans tous les wireframes des 5 espaces. Si Client REFERENCE est dans l'étape 04 du pipeline côté PRIVEOS Capital (Doc 3), il doit aussi apparaître dans le CRM de l'Ingénieur REFERENCE (Doc 5), dans la fiche cabinet (Doc 4), et dans son propre espace client (Doc 6). Toutes les références à ses données patrimoniales, son DCI, son étude, ses RDV doivent être identiques partout.

⭐ Conventions des noms-pivots
Client REFERENCE & Cliente REFERENCE = le fil rouge des wireframes (couple de cadres parisiens, étape 04 du pipeline)

Ingénieur REFERENCE = le héros des wireframes côté pro (Cabinet Paris Étoile)

Cabinet REFERENCE = synonyme de Cabinet Paris Étoile (le cabinet où exerce l'Ingénieur REFERENCE)

Le développeur peut chercher ces termes pour identifier rapidement les éléments-pivots.

📐 Règles de nommage des entités :
  • Identifiants techniques (IDs) : tous les enregistrements ont un ID unique au format type-XXXX (ex : cli-0001 pour client, cab-0001 pour cabinet, ing-0001 pour ingénieur, dos-0001 pour dossier, prd-0001 pour produit). Le développeur réutilisera ces IDs comme clé primaire.
  • Dates : toutes les dates sont au format YYYY-MM-DD. Les wireframes représentent l'état du système au 03 mai 2026 (date d'aujourd'hui).
  • Montants financiers : en euros, format français avec espaces fines (ex : 125 000 €) sauf mention contraire.
  • Pourcentages : format français avec virgule décimale (ex : 4,5 %).
  • Téléphones : format français (ex : 06 12 34 56 78).
  • Emails : tous les emails fictifs utilisent le domaine @email-test.fr pour éviter toute collision avec de vrais comptes.
✅ Règle de cohérence financière :

Les chiffres sont calculés et cohérents entre eux. Si Client REFERENCE a un encours de 120 000 € (somme de ses contrats), alors la somme de toutes les commissions générées par ce client doit correspondre, et la quote-part remontée à PRIVEOS Capital aussi. Le développeur peut utiliser ce dataset pour vérifier la justesse des calculs dans son code (KPIs, agrégats, totaux).
